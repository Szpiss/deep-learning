# 基于用户邻域的 Top-N 推荐作业说明

本次作业使用 `MovieLens 100K` 数据集，完成了一个典型的基于用户邻域的协同过滤推荐流程。

## 文件说明

- `实验代码.py`：主程序，包含数据集自动下载、数据划分、用户相似度计算、Top-N 推荐和效果评估。
- `数据集/`：程序第一次运行时会自动下载并解压 `ml-100k` 数据集到该目录。

## 实验流程

1. 读取 `u.data` 和 `u.item` 文件。
2. 将评分大于等于 4 的记录视为用户正反馈。
3. 按用户进行留一法划分：
   每个用户最后一次正反馈作为测试集，其余作为训练集。
4. 根据用户共同喜欢的物品构建用户相似度矩阵。
5. 选取与目标用户最相似的 `K` 个邻居用户。
6. 汇总邻居喜欢但目标用户未看过的电影，生成 Top-N 推荐列表。
7. 使用 `Precision`、`Recall`、`Coverage`、`Popularity` 指标评估推荐效果。

## 运行方式

在当前目录执行：

```bash
python3 实验代码.py
```

如果想调整参数，例如邻居数和推荐数量，可以执行：

```bash
python3 实验代码.py --topk 30 --topn 10 --sample-user 10
```

## 可写入实验报告的简短结论

基于用户邻域的 Top-N 推荐方法首先通过用户历史行为构建兴趣集合，再通过共同兴趣计算用户之间的相似度。对于目标用户，系统从最相似的邻居中选出其喜欢但目标用户尚未接触的物品，并按加权得分排序后生成推荐结果。该方法实现简单、可解释性强，适合作为协同过滤推荐系统的基础实验方案。
